JavaScript Logic

To harness the power of code we need to utilize programming logic to make dynamic decisions as our programs are executed. We'll start with conditional statement, with roughly follow the format:

        If (an expression which can be true or false){
            Then execute this block of code
        }

This is an 'if statement'. We'll break down the components in the next sections.

Evaluators

To use logic, we need to be able to evaluate one number/string/variable as it compares to another. For that we'll use basic evaluators you likely know from math class.

        > - greater than
        < -  less then
        >= -  greater than or equal to
        <= - less than or equal to

These are fairly straightforward evaluators. Though you'd likely feed at least one variable through this expression, it come down to types of logic like:

        If (8>5) {
            //Execute this code block
        }

There are a few more evaluators we need to be aware of for evaluating expressions:

        == - Equal to
        != - Not equal to
        === - Strict equal to
        !== - Strict not equal to

As we've previously discussed, datatypes in JavaScript can be pretty fluid. The programming language recognizes that the string "5" is actually equal to the number 5. So the strict evaluators also look at type.

        "3" == 3 //true
        "3" === 3 //false
        "" == false //true
        "" === false //false 
        null == undefined //true
        null === undefined //false

These are referred to as 'truthy' and 'falsy'. Falsy statements that would not satisfy the expression in an if statement are:

        var x = false;
        var x = 0;
        var x = "";
        var x = 10/'falsy';  //Becomes NaN
        var x; //not assigned a value (null or undefined).

And anything with a value would be considered truthy.

Since we use one the = sign to assign a value to a variable, that's why two (and three) equals signs are used to evaluate. And the exclamation point in many programming languages means 'not'. If we're evaluating a boolean statement: 

        if(myVariable == false){
            //Block of code
        }

would be the same as:

        if(!myVariable) {
            //Block of code
        }

if/else/else if

Now that we know the evaluators, we can make full if statements. There are three types. Let's set up a webpage with the appropriate HTML tags and start putting content into a script tag.

First, just one single if statement:

        <script type="text/javascript">
            if(8 > 4){
                alert("The 8 is bigger than the 4");
            }

            if("4" > 8){
                alert("The 4 is bigger than the 8");
            }

            alert("Done script");
        </script>

So, there are two conditional statements, one that is satisfied and one that is not. Try swapping out the numbers for Strings and booleans, and define a variable at the top of the script and use that in the statement.

So, if the if statement is satisfied, code is executed. Otherwise the program runs like nothing happened. But we will sometimes want something else to happen if that code is not satisfied, so we make an else statement.

        <script type="text/javascript">
            if(false === ""){
                alert("The 8 is bigger than the 4");
            } else {
                alert("The condition didn't apply");
            }

            alert("Done script");
        </script>

When our code gets complex, it starts to get confusing what expressions are satisfied and what are not, so if/else statements are a good way to make sure two blocks of code aren't executed, when there should only be one.

Finally, if we have a series of if-like conditions, but only one should be satisfied, we use an if/else if statement.

        <script type="text/javascript">
            if(8 > 10){
                alert("Expression one was met");
            } else if (11 > 10) {
                alert("Expression two was met");
            } else if (15 > 10) {
                alert("Expression three was met");
            } else {
                alert("No expression was met");
            }

            alert("Done script");
        </script>

Note that both the second and third expressions would have been satisfied. It will come up when multiple expressions may be satisfied, and you need to strategically organize your code.

Logical Operators

In many situations, we'll need our code to run differently based on a combination of factors. In our expressions, we insert the equivalent of 'and' and/or 'or' for that. && represents 'and', and || represents 'or'.

        <script type="text/javascript">
            if(12 > 10 && "20"==20){
                alert("Expression one was met");
            } 
               
            if (11 > 10 || 5 > 20) {
                alert("Expression two was met");
            } 
               
            var x = true;
              
            if (15 > 10 || 0 === false && !x) {
                alert("Expression three was met");
            } 

            alert("Done script");
        </script>

As you can see, evaluators and logical operators start to get pretty confusing, but they're crucial for figuring out how your program will execute.

Switch Statements

Switch statements are another way to make conditionals work. If statements and objects (learn about these next week) can also end up doing many of the same things, but as you know there's multiple ways to attack a problem in web development.

They're accomplished by giving the statement a parameter, and if it satisfies one of the options, it executes that block of code:

    <script type="text/javascript">
       var x = 2;
       
       switch(x) {
       
        case 1:
           alert("It was case one!");
           break;
       
        case 2:
           alert("It was case two!");
           break;
           
        case 3:
            alert("It was case three!");
            break;    
       }
       
       alert("Done script");
    </script>

For Loops

Along with if/else statements, the other most common logic statement is for loops. It's a block of code that will repeat until the conditions in the expression have been satisfied.

Here's an example to count to five:

    <script type="text/javascript">
        for (i = 0; i < 5; i++) {
            alert("This is time " + i + " through the loop");        
        }
        
        alert("done script");
    </script>

The first part of the for loop is the initialization which defines a variable that acts as counter, and can be used through the loop. The second part is the condition, which basically says 'as long as i is less than 5, go through the loop again. And finally, the third part is the update. In this loop it's adding one each time (i = i + 1), but it could be anything.

It's convenient when the loop utilizes the counter (i), but it doesn't have to. That can be there to merely keep track of the loop. But we can also change it around so it can satisfy numbers that aren't in succession.

    <script type="text/javascript">
        //Measuring inches for feet
        for (i = 0; i < 97; i = i + 12){
             var feet = i/12;
             console.log("The wall is " + i + " feet tall");
        }
    </script>

For Loops make it much easier to repeat blocks of code for different parameters and will definitely come in hands as we move forward.

There are two more loops that are used occasionally, the while loop and the do while loop. for loops can generally accomplish the same thing, but with the while loops, everything isn't initialized at the top of the loop:

    <script type="text/javascript">
        var i = 0;
           
        while(i < 5){
            alert("This is time " + i + "through the loop");
            i++;
        }
        alert("done script");    
    </script>

And the do while loop is only different since the condition is at the end, it will at least run once regardless if it's been satisfied:

     <script type="text/javascript">
        var i = 1;
           
        do {
            alert("This is time " + i + "through the loop");
            i ++;
        } while (i < 1);
        alert("done script");    
    </script>
