JavaScript Functions

We know how to declare, assign and use variables, whether it be Strings, numbers, booleans, or even arrays. We know how to change around those variables using arithmatic and concatenating. Now it's time to enclose all of this information in functions.

Functions are simply a statement or series of statements. It's not overly difficult to define. The key thing to remember is if you're writing code you want to be executed later in the program (as in after the page loads), you need a function.

A Basic Function

Let's start with a very basic function that will change some text on the page (not unlike what we've already done in class examples). Let's use Brackets, so we can see how the page responds as we add code.

In the HTML portion, besides the standard tags we need, type in the body:

    <h1>My Site</h1>  
    <div id="message">Welcome to this beautiful site!</div>

And then we add out JavaScript:

    <script type="text/javascript">
        function updateMessage(){
            var message = document.getElementById('message');
            message.innerHTML = "We're so happy you're here!";
        }
        
        alert('Into JavaScript');
    </script>

Now, nothing actually changed on our page, but the program ran through our JavaScript because the alert went off.

We declared our function called updateMessage(), but we need to call the function.

Diving into this function: it has the function keyboard, it has a function name. It doesn't have any parameters, which would be in the parenthesis. And in the curly statements is a block of code, which could be anything. For now it's relating to the DOM (document object model), and swapping out what's in the message div for the new sentence in quotes.

But we need to call this function. To do so, let's just add this below the alert in our script.

    updateMessage();

The function stores the block of code, to be used later. Calling the function executes the entire block of code.

Functions with Parameters

The above is good if we don't need to pass anything through the function (like button clicks). But sometimes we want to feed parameters through the function that are gathered from user input (like text fields) or other dynamic content (like time of day).

To use parameters in our functions, we need to put them in the parenthesis. To demonstate this, we'll use the prompt method. It's like the alert method, but gets user input. Let's replace the previous JavaScript with the following:

    <script type="text/javascript">
        var name = prompt("What is your name?");
        function updateMessage(messageName){
            var message = document.getElementById('message');
            message.innerHTML = "We're so happy you're here, " + messageName + "!";
        }
        
        updateMessage(name);
    </script>

The updateMessage(name); call sent the variable name as a parameter. When the function is receiving that parameter, it's just looking for one variable, and it can be named whatever inside of the function (I changed it in the above, but it could've been called 'name' still). We can also use as many parameters as we'd like.

Functions with Return Values

Sometimes we want functions to return information to the code that called them. We use return for that.

    <script type="text/javascript">
        function calculateArea(width, height){
            var area = width * height;
            return area;
        }
        
        var wallOne = calculateArea(3, 5);
        var wallTwo = calculateArea(8,5);
        
        alert("The area of the first wall is " + wallOne + ".";
        alert("The area of the second wall is " + wallTwo + ".";
    </script>

Return functions can also return multiple values, but we need to use an array for that.

    <script type="text/javascript">
        function getSize(width, height, depth){
            var area = width * height;
            var volume = width * height * depth;
            var sizes = [area, volume];
            return sizes;
        }
        
        var areaOne = getSize(3, 5, 3)[0];
        var volumeOne = getSize(8, 5, 10)[1];
        
        alert("The area of the first wall is " + areaOne + ".";
        alert("The volume of the second wall is " + volumeOne + ".";
    </script>

Anonymous Functions

The functions above are referred to as function declaration. They are declared and can be called later in code. The other type of functions are considered function expressions. They're placed where the interpreter of the web page would expect to see an expression (or assignment for a variable). They also don't have a name, which are referred to as Anonymous Functions.

The area function from above could be written as a function expressions as:

    var area = function(width, height) {
        return width * height;
    }
    
    var size = area(3, 5);

The function expression from above can be taken a step further and can be invoked immediately when the program runs that line of code.

    var area = (function(){
        width = 3;
        height = 5;
        return width * height;
    }());

The parenthesis near the end of the code () tell the interpreter to run the code immediately, and the parenthesis around the entire block of code make sure that it's considered to be one element.

Variable Scope

While our programs are very small, this doesn't become much of an issue, but once they get larger, it's important to properly keep track of variables. Variables can be either global or local. Global variables are defined at the start of the script section, and will remain relevant for the entire script. Local variables are defined within a function cannot be accessed outside of that function.

    <script type="text/javascript">
       
        var width = 10;
        var area;
        
        function getSize(){
            var height = 8;
            area = height * width;
        }
        
        getSize();
        alert("The width of the wall is " + width + ".");
        alert("The height of the second wall is " + height + ".");
        alert("The area of the wall is " + area + ".");
    </script>

It's important to note that if a variable is defined or assigned twice, the one that has the more local scope will be used.

    <script type="text/javascript">
       
        var width = 10;
        
        function getSize(){
            var height = 8;
            var width = 20;
            area = height * width;
        }
        
        getSize();
        alert("The width of the wall is " + width + ".";
        alert("The area of the wall is " + area + ".";
    </script>
